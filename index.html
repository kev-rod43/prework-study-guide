HTML

  • The head elemnt contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • All elements start with "element"and end with "/element", and both the start and end must be the same "space" from the index.
  • Child elements are denoted by being in a subsequent line, one space from the index fruther than the parent.
  • There are multiple elements, and attributes that can be attached to said elements, Cheat Sheet I Found Online

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • You can apply a css rule to an element directly, or you can crease a class attribute (."AttributeName") to be able to replicate the css rule to multiple different elements.
  • Color can be added to properties through either semantics or HTML color codes.
  • Gigantic reference list of CSS properties with indepth dive.

Git

  • "git status" checks what branch we are currently on
  • "git checkout -b branch-name" creates a new branch and switches to it
  • There are a magnitude of business best practices to maintain quality through personal actions such as:
    • Confirming what branch one is working on.
    • Confirming that our local main branch is the same version as the one in Github.
    • Committing often with meaningful comments to provide more control over changes.
    • Ensure that all PR's are reviewed by peers for QA, and provide proper description within the request.
